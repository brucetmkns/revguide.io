How to Export and Import Data

Back up your content and restore from backups

RevGuide allows you to export all your content to a JSON file for backup purposes, migration, or sharing. You can also import data from a backup file to restore content.

Who Can Export/Import

  • Export: Editors, Admins, and Owners
  • Import: Admins and Owners only

Exporting Data

1 Navigate to Settings

Open the RevGuide admin panel and click "Settings" in the left sidebar.

2 Find Import/Export Section

Scroll to the "Import/Export" card.

3 Click Export Data

Click the "Export Data" button. Your browser will download a JSON file.

4 Save the File

The file is automatically named with the date: revguide-backup-YYYY-MM-DD.json

What Gets Exported

The export file includes:

Content Type Included
Banners All banners with their settings and conditions
Plays (Battle Cards) All plays with categories and content
Wiki Entries All wiki entries including imported fields
Display Settings Banner position and visibility preferences

Not Included in Export

  • Team members and invitations
  • HubSpot connection details
  • Account credentials
  • User-specific preferences

Export File Format

The export is a JSON file with this structure:

{
  "version": "1.0",
  "exportedAt": "2024-01-15T10:30:00Z",
  "rules": [...],           // Banners
  "battleCards": [...],     // Plays
  "presentations": [...],   // Media items
  "wikiEntries": [...],     // Wiki entries
  "settings": {...}         // Display settings
}
Tip: Store export files in a secure location. They contain all your organization's RevGuide content.

Importing Data

Warning: Importing data replaces your existing content. Always export a backup before importing.
1 Export Current Data First

Create a backup of your current content before importing.

2 Navigate to Settings

Go to Settings and find the Import/Export section.

3 Click Import Data

Click the "Import Data" button. A file picker will open.

4 Select Your Backup File

Choose the RevGuide JSON backup file you want to import.

5 Review and Confirm

RevGuide will show you what will be imported:

  • Number of banners
  • Number of plays
  • Number of media items
  • Number of wiki entries

Confirm to proceed with the import.

6 Wait for Completion

The page will refresh automatically after import. All imported content will be immediately available.

Use Cases for Export/Import

Regular Backups

Schedule regular exports to protect against accidental deletion:

  • Weekly exports for active organizations
  • Monthly exports for stable content
  • Before making major changes

Content Migration

Move content between organizations:

  1. Export from the source organization
  2. Sign in to the destination organization
  3. Import the backup file

Staging/Testing

Test changes in a separate environment:

  1. Export production content
  2. Import to a test organization
  3. Make and test changes
  4. Export tested content
  5. Import back to production

Sharing Content

Share templates with other organizations:

  • Export your best banners and plays
  • Share the JSON file
  • Other organizations can import as a starting point

Troubleshooting

Invalid Backup File Error

  • Ensure the file is a valid JSON file
  • Check that it was exported from RevGuide
  • Verify the file wasn't corrupted during transfer
  • Try opening the file in a text editor to check for issues

Import Failed

  • Check your internet connection
  • Ensure you have Admin or Owner permissions
  • Try a smaller import if the file is very large
  • Contact support if the issue persists

Missing Content After Import

  • Import replaces all content - check your backup file contains everything
  • Verify you imported the correct file
  • Check if the content was disabled (toggle enabled status)
Note: Import is an all-or-nothing operation. You cannot selectively import specific items from a backup file. The entire file is imported at once.

Best Practices

  • Regular Exports: Create backups before major changes and on a regular schedule
  • Version Your Backups: Keep multiple dated backups, not just the latest
  • Test Imports: When migrating, test in a separate organization first
  • Document Changes: Note what's in each backup for easier recovery
  • Secure Storage: Store backups in a secure, accessible location
