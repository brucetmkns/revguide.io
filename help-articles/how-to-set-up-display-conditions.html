How to Set Up Display Conditions

Control where and when your content appears in HubSpot

Display conditions let you target specific HubSpot pages for your banners and wiki entries. Show the right content to the right people at the right time by using URL patterns and page types.

Understanding URL Patterns

RevGuide uses URL patterns to match HubSpot pages. Patterns can include wildcards (*) to match multiple pages.

Pattern Syntax

Pattern Matches Example URLs
* All pages Everything in HubSpot
/contacts/* All contact pages Contact records, contact lists
/contacts/*/record/* Contact record pages only Individual contact views
/deals/* All deal pages Deal records, pipeline board
/companies/* All company pages Company records, lists
/tickets/* All ticket pages Ticket records, queue

Common URL Patterns

CRM Object Pages

/contacts/*        - All contact pages
/contacts/*/record/* - Contact record views only
/companies/*       - All company pages
/deals/*           - All deal pages
/tickets/*         - All ticket pages

Specific Sections

/sales/*           - Sales tools
/marketing/*       - Marketing tools
/service/*         - Service tools
/reports/*         - Reports and analytics
/settings/*        - Settings pages

Pipeline and Board Views

/deals/board/*     - Deal pipeline board
/tickets/board/*   - Ticket board view

Setting Conditions for Banners

1 Open Banner Editor

Create a new banner or edit an existing one.

2 Find URL Pattern Field

In the banner settings, locate the URL pattern or display conditions section.

3 Enter URL Patterns

Enter one or more URL patterns, one per line. The banner will show on pages matching any of the patterns.

4 Test Your Banner

Navigate to a matching HubSpot page to verify your banner appears correctly.

Setting Conditions for Wiki Entries

Wiki entries have additional options in the Rules tab:

Page Type

Limit where tooltips appear:

  • Any: Show on all matching pages
  • Record: Only on individual record pages
  • List: Only on list views
  • Board: Only on board/pipeline views

URL Patterns for Wiki

Add URL patterns in the Rules tab to restrict where a specific tooltip appears. Leave empty to show everywhere.

Multiple Conditions

When you add multiple URL patterns:

  • Patterns work with OR logic
  • Content shows if ANY pattern matches
  • Add more patterns to include more pages
Tip: Start with broader patterns and narrow down based on user feedback. It's easier to remove content from specific pages than to figure out which pages you forgot to include.

Testing Your Conditions

  1. Create the condition: Set your URL pattern
  2. Save the content: Ensure it's enabled
  3. Navigate to HubSpot: Go to a page that should match
  4. Verify display: Check that content appears
  5. Test non-matching pages: Verify content doesn't appear where it shouldn't
Note: Changes take effect immediately. If you don't see your content, try refreshing the HubSpot page or clearing your browser cache.

Troubleshooting

Content Not Appearing

  • Verify the content is enabled (not disabled)
  • Check that your URL pattern matches the current page
  • Ensure there are no typos in the pattern
  • Try using a broader pattern like * to test

Content Appearing Everywhere

  • Check for overly broad patterns like *
  • Add more specific patterns to narrow the scope
  • Verify you haven't left the pattern field empty (which defaults to all pages)

Best Practices

  • Be Specific: Target the exact pages where content is relevant
  • Test Thoroughly: Check both matching and non-matching pages
  • Document Patterns: Use clear banner titles that indicate the target audience
  • Review Regularly: As HubSpot's URL structure may change, verify patterns still work
