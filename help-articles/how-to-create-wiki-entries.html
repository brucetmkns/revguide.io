How to Create Wiki Entries

Build a knowledge base with instant tooltips in HubSpot

Wiki entries are definitions and explanations that appear as tooltips when users hover over matching terms in HubSpot. Create entries for acronyms, industry terms, process definitions, field explanations, and any other knowledge your team needs.

Creating a Wiki Entry

1 Navigate to Wiki

Open the RevGuide admin panel and click "Wiki" in the left sidebar.

2 Click Add Entry

Click the "Add Entry" button. The entry editor will appear on the right side.

3 Fill in the Content Tab

Enter the following information:

  • Title: The display name of your entry
  • Trigger Word: The text that activates the tooltip (optional)
  • Aliases: Additional words that trigger this tooltip (comma-separated)
  • Definition: The explanation shown in the tooltip
  • Learn More Link: Optional URL for additional resources
4 Configure the Rules Tab

Set how and where the tooltip appears:

  • Category: Term type (General, Field, Process, etc.)
  • Match Type: Exact match or contains
  • Frequency: Show once per page or every occurrence
  • Priority: Tooltip precedence when terms overlap
5 Save Your Entry

Click "Save" to create the entry. It will be enabled by default.

Understanding Entry Fields

Title vs. Trigger Word

  • Title: The name shown in the admin panel and tooltip header. Always required.
  • Trigger Word: The text that activates the tooltip. Leave empty if you don't want a tooltip (documentation only).

Example: You might have a title "Marketing Qualified Lead" with trigger word "MQL".

Aliases

Aliases are alternative words that show the same tooltip. Enter multiple aliases separated by commas.

Example: For "Marketing Qualified Lead", aliases might be: MQL, Marketing-Qualified Lead, Marketing Qualified

Tip: Use aliases for common abbreviations, alternate spellings, and related terms that should show the same definition.

Categories

Organize entries by category:

Category Use For
General Business terms, acronyms, definitions
Field HubSpot property definitions
Process Workflow and procedure explanations
Product Product-related terminology
Competitor Competitor names and information

Match Type Options

Exact Match (Default)

Only matches the exact trigger word as a complete word. "MQL" won't match "MQLs" or "preMQL".

Contains

Matches the trigger word anywhere it appears. Use carefully to avoid unintended matches.

Frequency Options

Once Per Page (Default)

Only the first occurrence on each page gets a tooltip indicator. Reduces visual clutter.

Every Occurrence

Every instance of the trigger word gets a tooltip indicator. Use for critical terms users need to see everywhere.

Priority System

When multiple wiki entries could match the same text, priority determines which tooltip shows:

  • Higher priority numbers win (1-100 scale)
  • Default priority is 50
  • Use 75-100 for critical definitions
  • Use 25-49 for nice-to-have entries

Writing Good Definitions

Best Practices

  • Be Concise: Keep definitions brief and scannable
  • Front-load Information: Put the key point first
  • Use Formatting: Bold key terms, use lists for multiple points
  • Include Examples: Show how the term is used in context
  • Add Links: Reference detailed documentation when available

Example Definition

Term: MQL

Definition:

Marketing Qualified Lead - A lead that has engaged with marketing content and meets our qualification criteria.

MQL criteria:
- Downloaded 2+ content pieces
- Visited pricing page
- Company size 50+ employees

MQLs are automatically routed to Sales for follow-up within 24 hours.

Managing Wiki Entries

Bulk Operations

Use select mode to manage multiple entries:

  1. Click the checkbox icon to enter select mode
  2. Check entries you want to modify
  3. Use bulk actions to delete selected entries

Duplicate Entry Warning

RevGuide warns you if a trigger word or alias conflicts with an existing entry. Duplicate triggers can cause unpredictable tooltip behavior.

Note: Entries without a trigger word won't show tooltips but remain searchable in the wiki tree. Use this for documentation-only entries.

Organizing Entries

Wiki entries are organized in a tree structure:

  1. Object Type Level: Contacts, Companies, Deals, Tickets, Custom Terms
  2. Property Group Level: Groups from HubSpot or custom groupings
  3. Entry Level: Individual wiki entries

Use the filters and search to find entries quickly.
