How to Use Wiki Tooltips

Get instant definitions as you work in HubSpot

Wiki tooltips are contextual definitions that appear automatically when you encounter defined terms while working in HubSpot. They help your team understand terminology, field meanings, and processes without leaving their workflow.

How Tooltips Work

Automatic Detection

RevGuide scans HubSpot pages for text that matches your wiki entries. When a match is found:

  1. The matching text gets a subtle underline indicator
  2. Hovering over the text reveals the tooltip
  3. The tooltip shows the title, category, and definition

What Triggers a Tooltip

A tooltip appears when:

  • Text matches a wiki entry's trigger word
  • Text matches one of the entry's aliases (if enabled)
  • The wiki entry is enabled
  • The current page matches any URL pattern conditions

Example Tooltip

MQL General

Marketing Qualified Lead - A lead that has engaged with marketing content and meets qualification criteria.

MQL criteria: Downloaded 2+ content pieces, visited pricing page, company size 50+ employees.

Interacting with Tooltips

Viewing a Tooltip

1 Find Underlined Text

Look for text with a subtle dotted underline - this indicates a tooltip is available.

2 Hover to View

Move your mouse over the underlined text. The tooltip appears after a brief delay.

3 Read the Content

The tooltip stays visible while you hover. Move away to dismiss it.

Tooltip Elements

Element Description
Title The entry's display name
Category Badge Shows if it's a field, term, process, etc.
Definition The full explanation text
Learn More Link Optional link to additional documentation

Where Tooltips Appear

Tooltips can appear throughout HubSpot:

  • Record Pages: Contact, company, deal, and ticket records
  • Property Labels: Field names and descriptions
  • List Views: Column headers and cell values
  • Navigation: Menu items and section headers
  • Workflows: Workflow names and descriptions

Tooltip Display Options

Frequency Settings

How often the same tooltip appears on a page:

Setting Behavior Best For
Once per page Only first occurrence is underlined Most entries - reduces visual clutter
Every occurrence All instances are underlined Critical terms users need to see everywhere

Match Types

How text matching works:

Type Behavior Example
Exact Matches whole words only "MQL" matches, "MQLs" doesn't
Contains Matches anywhere in text "lead" matches "Leads", "unleaded"
Tip: Use "Exact" match for most entries to avoid false matches. Use "Contains" only when you want to catch variations.

Common Tooltip Use Cases

Acronyms and Abbreviations

Help team members understand company-specific acronyms:

  • MQL, SQL, SAL (lead stages)
  • ARR, MRR, ACV (revenue metrics)
  • CS, CSM, TAM (team roles)

HubSpot Field Definitions

Explain what fields mean and how to fill them:

  • Custom property definitions
  • Dropdown option explanations
  • Required vs. optional field guidance

Process Terminology

Define your company's specific processes:

  • Deal stages and exit criteria
  • Lead scoring rules
  • Handoff procedures

Product Terms

Keep everyone aligned on product language:

  • Feature names
  • Plan/tier names
  • Technical terms

Best Practices for End Users

Using Tooltips Effectively

  • Hover over unfamiliar terms - there may be a tooltip
  • Click "Learn More" links for detailed information
  • Use tooltips to learn standard terminology
  • Report missing definitions to your admin

When Tooltips Help Most

  • Onboarding: New team members learning systems
  • Data Entry: Understanding field meanings
  • Cross-team Work: Understanding other departments' terms
  • Sales Calls: Quick product/feature lookups

Requesting New Tooltips

If you encounter a term that should have a tooltip:

  1. Note the term and where you saw it
  2. Contact your RevGuide admin (check Settings for admins)
  3. Suggest a definition if you know what the term means
Note: Only Editors, Admins, and Owners can create new wiki entries. Viewers can request entries but cannot create them directly.

Troubleshooting Tooltips

Tooltip Not Appearing

  • Verify the term has a wiki entry with a trigger word
  • Check that the entry is enabled
  • The term must appear exactly as the trigger (for exact match)
  • Check page type restrictions on the entry
  • Refresh the HubSpot page

Wrong Tooltip Showing

  • Multiple entries may have overlapping triggers
  • Check aliases for conflicts
  • Higher priority entries take precedence
  • Report to admin to resolve conflicts

Too Many Tooltips

  • Admins can set entries to "once per page" frequency
  • Page type filters can limit where entries appear
  • Entries can be disabled temporarily

Privacy and Performance

Tooltips are designed to be lightweight:

  • Definitions are cached locally for speed
  • No data is sent when you view a tooltip
  • Page performance impact is minimal
  • All content is shared within your organization only
