How to Import HubSpot Fields to Wiki

Automatically document your HubSpot properties

RevGuide can import field definitions directly from your HubSpot portal, creating wiki entries for each property. This helps document your CRM schema and provides instant tooltips for field names throughout HubSpot.

Prerequisites

  • HubSpot portal connected to RevGuide (How to connect)
  • Admin or Owner role in your RevGuide organization

Import Steps

1 Navigate to Wiki

Open the RevGuide admin panel and click "Wiki" in the left sidebar.

2 Click Import Fields

Click the "Import Fields" button next to "Add Entry". The import modal will open.

3 Select Object Type

Choose which HubSpot object type to import fields from:

  • Contacts: Contact properties
  • Companies: Company properties
  • Deals: Deal properties
  • Tickets: Ticket properties
4 Review Available Fields

RevGuide will fetch all properties from HubSpot and display them grouped by property group. Each field shows:

  • Field label (display name)
  • Field API name
  • "Already added" badge for existing wiki entries
  • "Dropdown" badge for fields with options
5 Select Fields to Import

Check the boxes next to fields you want to import. You can:

  • Select individual fields
  • Select entire property groups with the group checkbox
  • Use "Select All" to import everything
  • Use the search box to filter fields
6 Import Dropdown Values (Optional)

Check "Import dropdown values as nested entries" to create child wiki entries for each option in dropdown/select fields.

7 Click Import Selected

Click the import button to create wiki entries for all selected fields.

What Gets Imported

Each imported field creates a wiki entry with:

Wiki Field Source
Title HubSpot field label
Trigger HubSpot field label (for tooltips)
Aliases HubSpot API name
Category Set to "Field"
Object Type Selected object (contacts, deals, etc.)
Property Group HubSpot property group name
Definition Auto-generated with field type and options

Importing Dropdown Values

When you enable "Import dropdown values as nested entries", RevGuide creates additional wiki entries for each option in dropdown fields:

  • Each option becomes a child entry linked to the parent field
  • Child entries include the option label, value, and description
  • Child entries appear nested under the parent in the wiki tree
  • Tooltips will appear for both field names and option values
Tip: Importing dropdown values is especially useful for fields like Lead Status, Lifecycle Stage, or custom picklists where users need to understand what each option means.

Updating Existing Entries

If you've previously imported a field, you'll see "Already added" badge. Selecting these fields again will update them with the latest property group information without overwriting your custom definitions.

Organizing Imported Fields

After import, your wiki entries are organized by:

  1. Object Type: Contacts, Companies, Deals, Tickets, or Custom Terms
  2. Property Group: The HubSpot property group (e.g., "Contact Information", "Deal Information")

Use the filters in the wiki navigation to view fields by object type or browse the tree structure.

Enhancing Imported Entries

Imported entries provide a starting point. Enhance them by:

  • Adding detailed definitions explaining how your team uses the field
  • Including examples of proper values
  • Adding links to related documentation
  • Setting custom aliases for common abbreviations
Note: Imported definitions are automatically generated. Review and customize them to provide more value to your team.

Troubleshooting

No fields appear

  • Verify your HubSpot connection is active in Settings
  • Check that you have permission to view properties in HubSpot
  • Try disconnecting and reconnecting HubSpot

Import button is disabled

  • Select at least one field to import
  • Ensure you're not only selecting "Already added" fields
